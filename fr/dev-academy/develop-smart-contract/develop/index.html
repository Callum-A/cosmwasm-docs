Développer le contrat

Dans cette session, nous allons nous appuyer sur le modèle que nous avons généré sur la page précédente.

Voici la logique de l'application que nous voulons :

  • Les jetons collectés dans le solde du contrat intelligent sont libérés à une adresse cible après que le montant des jetons dépasse un montant spécifié.
  • Le contrat accepte le jeton cw20 qui est prédéfini par l'administrateur.
warning

Nous recommandons de supprimer le code du modèle pendant l'implémentation pour faciliter le copier-coller du code.

Vous pouvez trouver la version complète de ce contrat sur cw-contracts/cw20-pot :::

Config​

Les informations admin qui déterminent qui contrôle l'adresse de création de pot doivent être persistées sur le stockage du contrat lors de chaque exécution. Nous enregistrerons ces informations lors de la phase de instantiate.

// state.rs
#[derive(Serialize, Deserialize, Clone, Debug, PartialEq, JsonSchema)]
pub struct Config {
  pub owner: Addr,
  pub cw20_addr: Addr
}

pub const CONFIG: Item<Config> = Item::new("config");

Item est utilisé car ces données seront un singleton.

Instantiate​

Msg​

// msg.rs
#[derive(Serialize, Deserialize, Clone, Debug, PartialEq, JsonSchema)]
pub struct InstantiateMsg {
    pub admin: Option<String>,
    /// cw20_addr is the address of the allowed cw20 token
    pub cw20_addr: String
}

admin est défini comme Option car si None, info.sender sera défini comme admin par défaut. L'adresse cw20 acceptée est définie ici.

Instantiate​

//contract.rs
#[cfg_attr(not(feature = "library"), entry_point)]
pub fn instantiate(
    deps: DepsMut,
    _env: Env,
    info: MessageInfo,
    msg: InstantiateMsg,
) -> Result<Response, ContractError> {
    set_contract_version(deps.storage, CONTRACT_NAME, CONTRACT_VERSION)?;

    let owner = msg.admin
        .and_then(|s| deps.api.addr_validate(s.as_str()).ok())
        .unwrap_or(info.sender);

    let config = Config {
        owner: owner.clone(),
        cw20_addr: deps.api.addr_validate(msg.cw20_addr.as_str())?
    } ;
    CONFIG.save(deps.storage, &config) ;

    // init pot sequence
    POT_SEQ.save(deps.storage, &Uint128::new(0))? ;

    Ok(Response::new()
        .add_attribute("method", "instantiate")
        .add_attribute("owner", owner)
        .add_attribute("cw20_addr", msg.cw20_addr))
}

La logique ci-dessus signifie que si le propriétaire est Some alors valider, sinon utiliser info.sender et enregistrer la configuration, puis retourner Response avec des attributs.

let owner = msg.admin
    .and_then(|s| deps.api.addr_validate(s.as_str()).ok())
    .unwrap_or(info.sender);

L'exemple ci-dessus est un excellent moyen de comprendre le chaînage des méthodes. Nous vous recommandons cette excellente lecture : Rust Combinator

Créer un pot commun​

admin peut créer des pots pour collecter de l'argent avec un seuil donné.

State​

// state.rs
#[derive(Serialize, Deserialize, Clone, Debug, PartialEq, JsonSchema)]
pub struct Pot {
  /// target_addr is the address that will receive the pot
  pub target_addr: Addr,
  /// threshold_amount is the token threshold amount
  pub threshold_amount: Uint128,
  /// collected keeps information on how much is collected for this pot.
  pub collected: Uint128,
}

/// POT_SEQ contient le dernier ID du pot commun
pub const POT_SEQ: Item<Uint128> = Item::new("pot_seq");
/// POTS sont indexés par un entier qui auto incrémente.
pub const POTS: Map<U128Key, Pot> = Map::new("pot");

Nous pouvons utiliser une fonction save_pot pour auto-incrémenter seq et l'utiliser comme index pour pot.

// state.rs
pub fn save_pot(deps: DepsMut, pot: &Pot) -> StdResult<()> {
  // increment id if exists, or return 1
  let id = POT_SEQ.load(deps.storage)?;
  // checks for overflow
  let id = id.checked_add(Uint128::new(1))?;
  POT_SEQ.save(deps.storage, &id)?;

  // save pot with id
  POTS.save(deps.storage, id.u128().into(), pot)
}

Msg​

ExecuteMsg devient ceci :

// msg.rs
#[derive(Serialize, Deserialize, Clone, Debug, PartialEq, JsonSchema)]
#[serde(rename_all = "snake_case")]
pub enum ExecuteMsg {
    CreatePot {
        /// target_addr will receive tokens when token amount threshold is met.
        target_addr: String,
        /// threshold est le montant du jeton pour la libération des jetons.
        threshold: Uint128
    },
}

Execute​

Depending on the message that is received Execute will either CreatePot or Receive:

// contract.rs
#[cfg_attr(not(feature = "library"), entry_point)]
pub fn execute(
    deps: DepsMut,
    _env: Env,
    info: MessageInfo,
    msg: ExecuteMsg,
) -> Result<Response, ContractError> {
    match msg {
        ExecuteMsg::CreatePot {
            target_addr,
            threshold,
        } => execute_create_pot(deps, info, target_addr, threshold),
        ExecuteMsg::Receive(msg) => execute_receive(deps, info, msg),
    }
}

L'exemple ci-dessus est un excellent exemple pour comprendre l'opérateur match. Maintenant que nous avons ajouté des options supplémentaires à ExecuteMsg nous allons revenir en arrière et mettre à jour l'enum.

//msg.rs
#[derive(Serialize, Deserialize, Clone, Debug, PartialEq, JsonSchema)]
#[serde(rename_all = "snake_case")]
pub enum ExecuteMsg {
    CreatePot {
        /// target_addr will receive tokens when token amount threshold is met.
        target_addr: String,
        /// threshold is the token amount for releasing tokens.
        threshold: Uint128,
    },
    /// Receive forwards received cw20 tokens to an execution logic
    Receive(Cw20ReceiveMsg),
}

Now that we have updated ExecuteMsg, we must create execute_create_pot which is called by the contract's Execute function.

// contract.rs
pub fn execute_create_pot(
    deps: DepsMut,
    info: MessageInfo,
    target_addr: String,
    threshold: Uint128,
) -> Result<Response, ContractError> {
    // owner authentication
    let config = CONFIG.load(deps.storage)?;
    if config.owner != info.sender {
        return Err(ContractError::Unauthorized {});
    }
    // create and save pot
    let pot = Pot {
        target_addr: deps.api.addr_validate(target_addr.as_str())?,
        threshold,
        collected: Uint128::zero(),
    };
    save_pot(deps, &pot)?;

    Ok(Response::new()
        .add_attribute("action", "execute_create_pot")
        .add_attribute("target_addr", target_addr)
        .add_attribute("threshold_amount", threshold))
}

Collecter des jetons​

C'est la partie importante que ce document tente d'enseigner : l'interaction avec un contrat externe. Le contrat intelligent collectera les jetons cw20. Après l'envoi du jeton cw20, ce contrat fonctionnera sur cette information.

Mais comment ?

Les contrats intelligents CosmWasm fonctionnent comme des acteurs qui envoient des messages. Chaque contrat exécute les autres en renvoyant un message au contexte.

Les utilisateurs peuvent transférer des jetons de leur compte vers le contrat intelligent, puis exécuter le contrat intelligent pour sauvegarder l'allocation de ce jeton dans la prochaine TX. Mais le problème ici est de savoir comment vérifier que ce jeton est envoyé par cet utilisateur ?

Une façon d'y parvenir :

  1. L'utilisateur augmente l'allocation de jetons de l'adresse du contrat intelligent cw20-pot.
  2. L'utilisateur déclenche le contrat cw20-pot pour retirer les fonds autorisés sur son compte.

Cette opération nécessite deux transactions.

Il existe un moyen meilleur et plus élégant : cw20 Receiver Interface.

Cela fonctionne comme suit : l'utilisateur crée un message pour envoyer des jetons cw20 au contrat cw20-pot avec un message intégré dans pour déclencher l'exécution du cw20-pot.

Si vous vérifiez cw20-base/contrats.rs#execute_send

pub fn execute_send(
  deps: DepsMut,
  _env: Env,
  info: MessageInfo,
  contract: String,
  amount: Uint128,
  msg: Binary,
) -> Result<Response, ContractError> {

Dans la signature, vous remarquerez contract, amount et msg. contract est le destinataire du jeton et aussi adresse de la prochaine exécution, amount est le nombre de jetons et msg est le message externe en base64.

À la fin de execute_send, vous verrez une Response avec un message incorporé renvoyé à la chaîne.

let res = Response::new()
        .add_attribute("action", "send")
        .add_attribute("from", &info.sender)
        .add_attribute("to", &contract)
        .add_attribute("amount", amount)
        .add_message(
            Cw20ReceiveMsg {
                sender: info.sender.into(),
                amount,
                msg,
            }
            .into_cosmos_msg(contract)?,
        );
    Ok(res)

Comme vous pouvez le voir, msg est enveloppé dans un Cw20ReceiveMsg et pour l'interaction avec le contrat cw20, votre contrat accepte Cw20ReceiveMsg.

pub fn execute_receive(
  deps: DepsMut,
  info: MessageInfo,
  wrapped: Cw20ReceiveMsg,
) -> Result<Response, ContractError> {
  // cw20 address authentication
  let config = CONFIG.load(deps.storage)?;
  if config.cw20_addr != info.sender {
    return Err(ContractError::Unauthorized {});
  }

  let msg: ReceiveMsg = from_binary(&wrapped.msg)?;
  match msg {
    ReceiveMsg::Send { id } => receive_send(deps, id, wrapped.amount, info.sender),
  }
}

Sur cette ligne, le message de réception enveloppé et défini par contrat est analysé à partir du binaire en base64.

  let msg: ReceiveMsg = from_binary(&wrapped.msg)?;

Voici le ReceiveMsg:

#[derive(Serialize, Deserialize, Clone, Debug, PartialEq, JsonSchema)]
#[serde(rename_all = "snake_case")]
pub enum ReceiveMsg {
    // Send sends token to an id with defined pot
    Send { id: Uint128 },
}

Cela crée une branche de deuxième profondeur à l'intérieur de execute.

Et le contrat intelligent exécute cette logique :

pub fn receive_send(
    deps: DepsMut,
    pot_id: Uint128,
    amount: Uint128,
    cw20_addr: Addr,
) -> Result<Response, ContractError> {
    // load pot
    let mut pot = POTS.load(deps.storage, pot_id.u128().into())?;

    pot.collected += amount;

    POTS.save(deps.storage, pot_id.u128().into(), &pot)?;

    let mut res = Response::new()
        .add_attribute("action", "receive_send")
        .add_attribute("pot_id", pot_id)
        .add_attribute("collected", pot.collected)
        .add_attribute("threshold", pot.threshold);

    // if collected exceeds threshold prepare a cw20 message
    if pot.collected >= pot.threshold {
        // Cw20Contract is a function helper that provides several queries and message builder.
        let cw20 = Cw20Contract(cw20_addr);
        // Build a cw20 transfer send msg, that send collected funds to target address
        let msg = cw20.call(Cw20ExecuteMsg::Transfer {
            recipient: pot.target_addr.into_string(),
            amount: pot.collected,
        })?;
        res = res.add_message(msg);
    }

    Ok(res)
}

Après l'exécution, si le seuil est dépassé, le montant collecté est envoyé à la cible.
