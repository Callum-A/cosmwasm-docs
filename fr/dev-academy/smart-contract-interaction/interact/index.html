Notions de base sur les interactions des contrats intelligents

Comme indiqué précédemment, les contrats intelligents sont des codes exécutables. Dans les prochaines leçons, nous apprendrons à en écrire un. En attendant, il est préférable d'utiliser ce qui est déjà écrit pour garder les choses simples. Nous allons plonger dans deux options pour déployer et interagir avec les contrats : wasmd et CosmJS.

Où trouver des contrats intelligents ?​

Normalement, nous compilons les contrats intelligents en utilisant des compilateurs rust puis des optimiseurs de code. Maintenant, nous allons simplement télécharger une version précompilée par l'équipe cosmwasm.

Télécharger​

Nous fournissons l'exécutable binaire de smart contract à cw-plus repo à côté du code. Le repo cw-plus est une collection de contrats intelligents de qualité production qui a été largement testée sur des réseaux réels. Vous verrez une liste des contrats disponibles sur la page du référentiel. Allez cliquer sur le bouton Releases pour voir les exécutables binaires étiquetés. Vous pouvez télécharger des binaires et les déployer sur les blockchains compatibles.

Nous utiliserons le binaire précompilé cw20 pour ce cours : cw20-base

Ne prêtez pas attention aux détails de la cw20-base pour le moment, concentrez-vous sur le deploiement d'un contrat sur un testnet.

wasmd​

Store (base de données)​

Nous allons déployer le code en utilisant wasmd CLI que nous avons installé plus tôt.

curl -LJO https://github.com/CosmWasm/cw-plus/releases/download/v0.8.0/cw20_base.wasm
RES=$(wasmd tx wasm store cw20_base.wasm --from wallet $TXFLAG -y)

# obtenir l'identifiant du code
CODE_ID=$(echo $RES | jq -r '.logs[0].events[1].attributes[0].value')

# imprimer l'identifiant du code
echo $CODE_ID

# pas encore de contrat, ceci devrait retourner une liste vide
wasmd query wasm list-contract-by-code $CODE_ID $NODE --output json

Maintenant le code stocké sur le réseau. CODE_ID est l'identifiant du code.

Nous avons maintenant un contrat instancié prêt à être utilisé. Comme vous pouvez le constater, vous avez besoin de beaucoup de manipulation de JSON dans le shell pour l'interaction avec la ligne de commande. C'est juste du travail fastidieux... Heureusement, nous avons une meilleure option.

CosmJS​

CosmJS est le couteau suisse qui permet de faire fonctionner des solutions client basées sur JavaScript, qu'il s'agisse d'applications/explorateurs Web, d'extensions de navigateur ou de clients côté serveur comme les faucets/scrapers dans l'écosystème Cosmos.

CosmJS contient toutes les fonctions dont vous avez besoin pour interagir avec les clients du SDK Cosmos. Son utilisation va des échanges, des explorateurs de blocs aux scripts de base, y compris l'exécution de contrats intelligents.

Pour ce tutoriel, nous allons explorer le côté CosmWasm de CosmJS.

Configuration du client​

La première étape avant toute chose est de s'assurer que nous pouvons créer un compte et nous connecter à la chaîne. Vous utiliserez toujours la commande suivante pour démarrer le @cosmjs/cli avec quelques aides préchargées de cw20-specific (en plus de toutes les aides générales qu'il possède).

npx @cosmjs/cli@^0.26 --init https://raw.githubusercontent.com/InterWasm/cw-plus-helpers/main/base.ts --init https://raw.githubusercontent.com/InterWasm/cw-plus-helpers/main/cw20-base.ts

Maintenant vous devez voir un client REPL.

const [addr, client] = await useOptions(cliffnetOptions).setup('password');
client.getAccount(addr);

Cela prendra quelques secondes car nous frappons le robinet la première fois pour nous assurer que vous avez des jetons sur votre compte pour payer les frais de gas. Quand il revient, vous devriez voir quelque chose comme ceci :

  address: 'wasm1w740h56j9nhudykkm80j5rf6ms25nhe9huuvgp',

    type: 'tendermint/PubKeySecp256k1',

Recharger son portefeuille​

Vous pouvez continuer à écrire dans le shell, ou le fermer et exécuter certaines sections plus tard. Commencez toujours par :

const [addr, client] = await useOptions(cliffnetOptions).setup(PASSWORD_HERE);

pour configurer votre client. useOptions takes the cliffnet configuration from everything from URLs to tokens to bech32prefix. When you call setup with a password, it checks for ~/.cliffnet.key and creates a new key if it is not there, otherwise it loads the key from the file. Votre clé privée (en fait mnémonique) est stockée de manière cryptée, et vous avez besoin du même mot de passe pour l'utiliser à nouveau. Try cat ~/.cliffnet.key to prove to yourself that it is indeed encrypted, or try reloading with a different password.

Si vous voulez le mnémonique, vous pouvez le récupérer à tout moment, pour autant que vous ayez toujours le fichier et le mot de passe. You could use this later to recover, or use the same mnemonic to import the key into the cliffnet cli tool.

useOptions(cliffnetOptions).recoverMnemonic(YOUR_PASSWORD_HERE)
This command saves the key to ~/.cliffnet.key encrypted. Si vous oubliez le mot de passe, supprimez-le ou passez un filename avec un mot de passe pour créer une nouvelle clé. :::

Essayez aussi avec un mot de passe invalide et voyez l'erreur que cela produit.

Maintenant que vous vous sentez un peu plus sûr de vos clés (et de votre capacité à les charger plus tard), commençons à travailler avec certains contrats.

Exemple : STAR​

The first contract I uploaded was STAR tokens, or "Golden Stars" to be distribute to the first 3 validators on the network.

Ne copiez pas ce texte mot pour mot, mais regardez comment un tel contrat est mis en place et déployé la première fois.

const [addr, client] = await useOptions(cliffnetOptions).setup(PASSWORD_HERE);

const cw20 = CW20(client, cliffnetOptions.fees);
const codeId = await cw20.upload(addr);
console.log(`CodeId: ${codeId}`);
// output: 55

// enable REPL editor mode to edit multiline code then execute

const initMsg = {
  name: "Golden Stars",
  symbol: "STAR",

    {address: "wasm1ppgpwep3yzh8w3d89xlzlens3420j5vs5q3p4j", amount: "10000"},
    {address: "wasm1fnx5jzqsdkntlq2nspjgswtezf45u5ug3kq9sw", amount: "10000"},

Spin Nouveau Contrat​

Maintenant que nous l'avons mis en ligne, nous pouvons facilement établir un second contrat. Cette fois-ci, veuillez vérifier et personnaliser les noms des champs et les montants des jetons avant de les saisir.

const [addr, client] = await useOptions(cliffnetOptions).setup(PASSWORD_HERE);
const cw20 = CW20(client, cliffnetOptions.fees);

.editor
const initMsg = {
  name: "My Coin",
  symbol: "MINE",
  decimals: 6,
  initial_balances: [
    {address: addr, amount: "12345678000"},
  ],
  mint: {
    minter: addr,
    cap: "99900000000"
  },
};
^
D

const codeId = 55;
const mine = await cw20.instantiate(addr, codeId, initMsg, "MINE");
console.log(`Contract: ${mine.contractAddress}`);
// Contract: wasm10ajume5hphs9gcrpl9mw2m96fv7qu0q7esznj2

// now, check the configuration
mine.balance(addr);
mine.tokenInfo()
mine.minter()

Regardez, vous êtes riches maintenant ! Il est temps de partager la richesse.

Utilisation de contrat​

Dans cette section, nous allons vous montrer comment utiliser votre jeton nouvellement construit. Vous pouvez continuer à taper dans le même REPL que vous avez utilisé pour créer les jetons MINE (ou tout autre nom plus approprié que vous avez inventé), mais si vous l'avez fermé et que vous revenez, voici comment vous reconnecter :

const [addr, client] = await useOptions(cliffnetOptions).setup(PASSWORD_HERE);
const cw20 = CW20(client, cliffnetOptions.fees);

// if you forgot your address, but remember your label, you can find it again
const contracts = await client.getContracts(55)
contracts
const contractAddress = contracts.filter(x => x.label === 'MINE')[0].address;

// otherwise, you can just cut and paste from before
const contractAddress = "wasm14wm5jvsm6r896tcqsx9dlxc8h0w2mg5de39dsm"

// now, connect to that contract and make sure it is yours
const mine = cw20.use(contractAddress);
mine.tokenInfo()
mine.minter()
mine.balance(addr)

Ok, vous êtes connecté à votre contrat. Voyons voir de quoi est capable cw20. Ici, je vais vous montrer comment vous pouvez frapper des jetons (vous vous êtes donné cette permission spéciale lors de l'init, n'est-ce pas ?) et transférer des jetons à d'autres utilisateurs.

const someone = "wasm13nt9rxj7v2ly096hm8qsyfjzg5pr7vn56p3cay";
const other = "wasm1ve2n9dd4uy48hzjgx8wamkc7dp7sfdv82u585d";

// right now, only you have tokens
mine.balance(addr)
mine.balance(someone)
mine.balance(other)
// and watch the total
mine.tokenInfo()

// let's mint some tokens for someone
mine.mint(addr, someone, "999888000")
// Bonus, take the tx hash printed out and cut-paste that into https://bigdipper.wasmnet.cosmwasm.com
// eg 26D5514CF437EE584793768B56CB4E605F1F6E61FC0123030DC64E08E2EE97FA

// See balances updated
mine.balance(someone)
mine.balance(addr)
// and the supply goes up
mine.tokenInfo()

// Okay, now let's transfer some tokens... that is the more normal one, right?
mine.transfer(addr, other, "4567000");
// eg. 4A76EFFEB09C82D0FEB97C3B5A9D5BADB6E9BD71F4EF248A3EF8B232C2F7262A
mine.balance(other)
mine.balance(addr)
